# Contributing to Modding 101

Thank you for your interest in contributing to Modding 101! This document provides guidelines for contributing to this project.

## How to Contribute

1. **Fork the repository** on GitHub
2. **Create a new branch** for your changes (`git checkout -b feature/your-feature-name`)
3. **Make your changes** and commit them with clear, descriptive messages
4. **Push to your fork** (`git push origin feature/your-feature-name`)
5. **Submit a Pull Request** describing your changes

## Contribution Guidelines

- **Keep tutorials beginner-friendly** - Explain concepts clearly
- **Include code examples** - Use proper syntax highlighting
- **Test your tutorials** - Ensure they work as described
- **Add comments** - Help others understand the code
- **Follow the existing structure** - Keep tutorials organized

## Code of Conduct

- Be respectful and inclusive
- Help others learn
- Provide constructive feedback
- No spam or self-promotion

## Questions?

Feel free to open an issue if you have questions or suggestions!
